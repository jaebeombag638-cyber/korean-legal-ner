# Korean Smishing Detector — 프로젝트 정리

## 프로젝트 개요

AI Hub 한국어 SMS 데이터를 활용해 스미싱/정상 문자를 이진 분류하는 머신러닝 프로젝트.

- **v1 (현재)**: ML 기반 베이스라인 구축, 품질 집중
- **v2 (추후)**: KoBERT 딥러닝 모델로 한계 극복 (별도 레포)

---

## 전체 흐름

```
원문 텍스트 → KoNLPy(형태소 분리) → TF-IDF(숫자 행렬) → ML 모델(학습/예측) → matplotlib/seaborn(시각화)
```

---

## 기술스택별 역할

### pandas
**역할: 데이터 테이블 관리자**

CSV 파일을 DataFrame(표)으로 불러와 탐색·전처리하는 도구.

```python
df = pd.read_csv('smishing_data.csv')   # 데이터 로딩
df['label'].value_counts()              # 스미싱/정상 개수 확인
df.dropna(subset=['text'])              # 결측값 제거
df['text']                              # 문자 내용 컬럼 추출
```

이 프로젝트에서 데이터 전처리 전반을 담당. 비중이 가장 높음.

---

### numpy
**역할: 숫자 배열 계산기 (scikit-learn의 뒷받침 엔진)**

직접 쓸 일은 거의 없고, scikit-learn이 내부적으로 사용. TF-IDF 결과가 numpy 배열 형태로 나옴.

```python
np.array([1, 0, 1, 1, 0])   # 라벨 배열
np.mean(scores)              # 점수 평균
```

---

### KoNLPy (Okt)
**역할: 한국어 단어 쪼개기**

한국어는 조사·어미가 단어에 붙어있어 단순 띄어쓰기로 분리가 안 됨. Okt가 형태소 단위로 분리하고 명사만 추출.

```python
okt.morphs("계좌로 입금해주세요")
# ['계좌', '로', '입금', '해', '주세요']

okt.nouns("계좌로 입금해주세요")
# ['계좌', '입금']
```

어미가 달라도 명사는 동일하게 추출되므로 TF-IDF가 같은 단어로 인식 가능.

---

### TF-IDF
**역할: 텍스트 → 숫자 변환 (단어 중요도 점수)**

ML 모델은 문자열을 직접 읽지 못하므로 숫자 행렬로 변환 필요.

- **TF (단어 빈도)**: 이 문자에서 이 단어가 얼마나 자주 나왔나
- **IDF (역문서 빈도)**: 이 단어가 전체 문자들에서 얼마나 희귀한가
- **TF-IDF = TF × IDF**: 특정 문자에만 자주 등장하는 단어에 높은 점수

```
문자 A: "무료 상품권 클릭 입금 계좌"  → 스미싱 키워드 점수 높음
문자 B: "오늘 저녁 뭐 먹을래 연락해"  → 일상 단어, 점수 낮음
```

### scikit-learn
**역할: ML 파이프라인 전체 (TF-IDF 벡터화 → 학습 → 평가)**

**TF-IDF 벡터화:**
```python
vectorizer = TfidfVectorizer()
X = vectorizer.fit_transform(texts)
```

**모델 학습:**


```python
print(classification_report(y_test, y_pred))

```

**비교할 3가지 모델:**

| 모델 | 특징 |
|------|------|
| Logistic Regression | 기준선. 단순하지만 텍스트 분류에 강함 |
| Naive Bayes | 스팸 필터 전통 방식. 빠르고 가벼움 |
| SVM | 고차원 데이터(TF-IDF 행렬)에 효과적 |

## 진행 단계별 설명

### 0단계: 준비
- AI Hub에서 스미싱/스팸 SMS 데이터셋 다운로드
- `data/` 폴더에 저장

---

### 1단계: 데이터 탐색 (EDA)

**목적:** "이 데이터가 어떻게 생겼나" 파악

```python
df.head()                        # 처음 5행 보기
df.shape                         # 몇 행 몇 열?
df.info()                        # 컬럼 타입, 결측값 확인
df['label'].value_counts()       # 스미싱/정상 개수 비율
df['text'].str.len().describe()  # 문자 길이 분포
```

EDA에서 발견한 것(클래스 불균형, 결측값, 길이 패턴)이 이후 전처리 방식을 결정.
포트폴리오에서 시각화가 "데이터를 이해하고 시작했다"는 걸 보여줌.

---

### 2단계: 전처리

**목적:** 모델이 학습할 수 있는 숫자 형태로 변환

```python
# 1. 결측값 제거
df = df.dropna(subset=['text'])

# 2. KoNLPy로 명사 추출
def extract_nouns(text):
    return ' '.join(okt.nouns(text))
df['nouns'] = df['text'].apply(extract_nouns)

# 3. TF-IDF 벡터화
vectorizer = TfidfVectorizer(max_features=5000)
X = vectorizer.fit_transform(df['nouns'])

# 4. 학습/테스트 분리 (80:20)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)
```

| 작업 | 이유 |
|------|------|
| 결측값 제거 | 빈 문자가 있으면 KoNLPy 오류 |
| 명사 추출 | 어미 제거해야 같은 단어로 인식 |
| TF-IDF | 모델은 숫자만 읽음 |
| 학습/테스트 분리 | 본 적 없는 데이터로 성능 측정해야 의미 있음 |

---

### 3단계: 모델 학습·비교

**목적:** 3개 모델 학습 후 성능 비교 → 최종 선택 근거 마련

```python
models = {
    'Logistic Regression': LogisticRegression(),
    'Naive Bayes': MultinomialNB(),
    'SVM': LinearSVC()
}

for name, model in models.items():
    model.fit(X_train, y_train)
    score = model.score(X_test, y_test)
    print(f'{name}: {score:.4f}')
```

모델 하나만 쓰면 "좋은지 나쁜지" 기준이 없음. 비교가 곧 포트폴리오의 핵심 스토리.

---

### 4단계: 성능 평가

**목적:** accuracy 하나로 부족한 이유 → F1·Confusion Matrix로 깊이 분석

accuracy만 보면 안 되는 이유:
```
전체 1000개 중 스미싱 10개면 "다 정상"이라고만 해도 accuracy 99% → 의미없는 모델
스미싱 탐지에서 중요한 건 "스미싱을 놓치지 않는 것" → Recall이 핵심
```

```python
print(classification_report(y_test, y_pred))  # Precision / Recall / F1
sns.heatmap(confusion_matrix(y_test, y_pred), annot=True, fmt='d')

# 오분류 사례 분석 (포트폴리오 깊이를 보여주는 부분)
wrong = df[(y_test != y_pred) & (y_test == '스미싱')]
```

---

### 5단계: 결론

**목적:** "이 프로젝트로 뭘 알게 됐나" 정리. 포트폴리오에서 가장 읽히는 부분.

- 모델별 성능 비교표
- 최종 모델 선택 이유 (숫자 + 한 문단 설명)
- 한계점 (신조어/은어, 특정 시기 데이터 편향, KoNLPy 속도)
- v2 방향: KoBERT로 문맥 이해 모델 확장

한계점을 쓰는 게 약점 노출이 아님. "모델을 맹목적으로 믿지 않고 비판적으로 이해한다"는 신호로 면접관이 주목함.

## 전체 흐름 요약

```
0. 준비     → AI Hub 데이터 다운로드
1. EDA      → 데이터 구조·분포 파악, 키워드 시각화
2. 전처리   → KoNLPy → TF-IDF → 학습/테스트 분리
3. 모델 학습 → 3개 모델 학습 및 accuracy 비교
4. 성능 평가 → F1, Confusion Matrix, 오분류 분석
5. 결론     → 모델 선택 근거, 한계점, v2 방향
```
