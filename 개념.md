## NER (Named Entity Recognition, 개체명 인식)

텍스트에서 "의미 있는 고유 명사"를 찾아내서 분류하는 작업.

### 예시

```
"피고 최도선은 2021년 3월 부산시 중구에서 사건을 일으켰다."
```

NER 결과:

| 텍스트 | 태그 | 의미 |
|--------|------|------|
| 최도선 | PER | 사람 (Person) |
| 2021년 3월 | DAT | 날짜 (Date) |
| 부산시 중구 | LOC | 장소 (Location) |

### 이 프로젝트에서 NER의 역할

익명화를 자동화하려면 먼저 **어디에 개인정보가 있는지 탐지**해야 한다. 그게 NER의 역할이다.

```
탐지 (NER)               →   교체 (익명화)
"최도선"    = PER        →   "증인-1"
"부산시 중구" = LOC      →   "부산광역시 중구 A"
"2021년 3월" = DAT       →   "2021년 XX월"
```

NER이 틀리면 (못 찾거나 잘못 분류하면) 익명화도 틀린다. 전체 성능이 NER 정확도에 달려 있다.

### 왜 KLUE-BERT를 쓰는가

BERT는 문장 전체 문맥을 보고 판단한다. 단순 규칙(예: "2글자면 이름")보다 훨씬 정확하다.

예: "이사"가 사람 이름인지 행위인지 — 앞뒤 문맥 없이는 판단 불가. BERT는 문맥을 보고 구분한다.

## BIO 태그

NER 결과를 텍스트에 붙이는 표기 방식. 토큰(단어 조각)마다 태그를 하나씩 붙인다.

### B, I, O 의미

| 태그 | 의미 | 언제 붙이나 |
|------|------|------------|
| **B**-PER | Begin | 개체명의 **첫 번째** 토큰 |
| **I**-PER | Inside | 개체명의 **두 번째 이후** 토큰 |
| **O** | Outside | 개체명이 **아닌** 토큰 |

### 예시

```
문장: "피고 최도선은 부산시에 살았다"

토큰:  피고   최     ##도   ##선  ##은  부산   ##시  ##에  살   ##았  ##다
태그:   O    B-PER  I-PER  I-PER   O   B-LOC  I-LOC   O    O    O    O
```

### 왜 B와 I를 구분하나

구분 안 하면 붙어있는 두 개체를 하나로 착각한다.

```
"홍길동 김철수" (사람 2명)

구분 없을 때:  PER PER PER PER PER       → 하나로 읽힘  ❌
BIO 사용 시:   B-PER I-PER I-PER B-PER I-PER  → 두 개로 읽힘  ✓
```

### 이 프로젝트에서 BIO가 만들어지는 흐름

```
JSON span 정보
    ↓
char_labels 배열 (문자 하나하나에 B/I/O 부여)
    ↓
BERT 토크나이저로 subword 분리
    ↓
token_labels 배열 (토큰 하나하나에 정수 레이블 부여)
    ↓
모델 학습 입력
```

`section_to_bio` 함수가 이 흐름 전체를 담당한다.

## Subword 토크나이저

### 왜 단어 단위로 자르지 않나

단어 단위로 자르면 모르는 단어가 너무 많아진다.

```
"최도선"  → 사전에 없음 → [UNK]  ❌
"홍길동은" → 사전에 없음 → [UNK]  ❌
```

한국어는 조사·활용 변형이 많아 모든 형태를 사전에 담는 게 불가능하다.

### Subword 방식

자주 나오는 조각은 그대로 두고, 드문 단어는 더 작은 조각으로 쪼갠다.

```
"최도선"  → ["최", "##도", "##선"]
"부산시"  → ["부산", "##시"]
"판결문"  → ["판결", "##문"]
"피고"    → ["피고"]           ← 자주 나오니 그대로
```

`##`은 "이 조각은 앞 조각에 이어붙는다"는 표시다.

### BIO 태그와의 연결

"최도선"이 3개 토큰으로 쪼개지면 태그를 3개 붙여야 한다.

```
토큰:    최         ##도        ##선
offset: (259, 260)  (260, 261)  (261, 262)
```

코드는 `offset_mapping`으로 각 토큰의 시작 문자 위치를 알아내고, 그 위치의 `char_labels`를 가져온다.

```python
# char_labels[259] = "B-PER"  (span 시작)
# char_labels[260] = "I-PER"
# char_labels[261] = "I-PER"

for token_start, token_end in encoding['offset_mapping']:
    token_labels.append(label2id[char_labels[token_start]])

# 결과:
# "최"   → char_labels[259] → B-PER ✓
# "##도" → char_labels[260] → I-PER ✓
# "##선" → char_labels[261] → I-PER ✓
```

### 특수 토큰 [CLS], [SEP]

BERT는 문장 앞뒤에 특수 토큰을 자동으로 붙인다.

```
입력:  최도선은 부산시에 살았다
토큰: [CLS] 최 ##도 ##선 ##은 부산 ##시 ##에 살 ##았 ##다 [SEP]
```

이 토큰들은 실제 텍스트가 아니라 `offset = (0, 0)`으로 표시된다. 코드에서 `-100`을 붙여 loss 계산에서 제외한다.

```python
if token_start == 0 and token_end == 0:
    token_labels.append(-100)   # [CLS], [SEP] → 학습 무시
```

## 기술 스택

### 언어 모델 — KLUE-BERT (klue/bert-base)

#### 언어 모델이란

텍스트를 입력하면 각 단어가 문장에서 어떤 의미인지 숫자(벡터)로 변환해주는 프로그램이다.

```
"최도선은 부산에 살았다"
    ↓ 언어 모델
최도선 → [0.2, 0.8, 0.1, ...]  ← 이 숫자 패턴이 "사람 이름"을 암시
부산   → [0.9, 0.1, 0.3, ...]  ← 이 숫자 패턴이 "장소"를 암시
```

NER은 이 숫자를 보고 `최도선 = PER`, `부산 = LOC` 로 분류한다.

#### BERT란

Google이 2018년 만든 언어 모델. 위키피디아 같은 대용량 텍스트를 미리 학습해서 언어의 패턴을 이미 알고 있다.

이걸 **파인튜닝(fine-tuning)** 방식으로 우리 데이터에 추가 학습시킨다.

```
[BERT 사전학습]          [파인튜닝]
수억 개 문장 학습   →   판결문 NER 데이터로 추가 학습
(이미 완료됨)            (우리가 하는 작업)
```

처음부터 만들지 않아도 되므로 데이터가 적어도 잘 동작한다.

#### 왜 KLUE-BERT인가

BERT는 영어로 사전학습됐다. 한국어에 쓰면 성능이 낮다.

**KLUE(Korean Language Understanding Evaluation)** 는 카카오 등이 만든 한국어 전용 BERT다.

| | 영어 BERT | KLUE-BERT |
|---|---|---|
| 사전학습 데이터 | 영어 위키 등 | 한국 뉴스, 위키, 법률문서 등 |
| "판결" 이해 | 어색 | 자연스러움 |
| NER 기준 성능 | 낮음 | 높음 |

> KLUE-BERT = "한국어를 이미 잘 아는 110M짜리 뇌". 판결문 NER 데이터로 추가 학습시키는 것이 이 프로젝트의 핵심이다.

### 학습 프레임워크 — HuggingFace Transformers + PyTorch

#### PyTorch란

딥러닝 계산을 해주는 엔진이다. 신경망의 숫자 계산(행렬 곱, 역전파 등)을 GPU에서 빠르게 처리한다.

```
KLUE-BERT의 110M 파라미터 → 전부 PyTorch 텐서(숫자 배열)로 저장됨
학습 = PyTorch가 이 숫자들을 조금씩 조정하는 과정
```

#### HuggingFace Transformers란

PyTorch로 BERT를 직접 구현하면 수천 줄이다. HuggingFace는 이걸 몇 줄로 쓸 수 있게 포장해놓은 라이브러리다.

```python
from transformers import AutoModelForTokenClassification
model = AutoModelForTokenClassification.from_pretrained("klue/bert-base")
```

`from_pretrained("klue/bert-base")` 한 줄로 KLUE-BERT의 사전학습된 가중치를 다운받아 바로 쓴다.

#### 둘의 관계

```
HuggingFace Transformers  +  PyTorch
       (포장지)                (엔진)
          ↓
   KLUE-BERT 불러오기, 학습, 예측 — 전부 여기서 처리
```

### 평가 — seqeval

#### 왜 일반 정확도(accuracy)를 안 쓰나

NER은 단어 하나하나가 아니라 **엔티티 전체**가 맞아야 의미 있다.

```
정답:   최 / 도 / 선  →  B-PER / I-PER / I-PER
예측:   최 / 도 / 선  →  B-PER / I-PER / O      ← 마지막 하나 틀림
```

단어 3개 중 2개 맞으니 accuracy = 66%  
하지만 `최도선`이라는 사람 이름을 못 잡은 것이므로 실제론 완전 실패다.

#### seqeval이 계산하는 것: entity-level F1

엔티티 **전체**가 정확히 일치할 때만 정답으로 친다.

| 지표 | 의미 |
|---|---|
| Precision | 모델이 PER이라고 한 것 중 실제 PER 비율 |
| Recall | 실제 PER 중 모델이 잡아낸 비율 |
| **F1** | Precision과 Recall의 조화평균 |

```python
from seqeval.metrics import f1_score
f1_score(y_true, y_pred)  # BIO 태그 리스트를 넣으면 entity-level F1 반환
```

> seqeval = "엔티티 전체가 맞아야 정답" 기준으로 F1을 계산하는 NER 전용 평가 도구.

### 데모 앱 — Streamlit + Hugging Face Spaces

#### Streamlit이란

모델 결과를 보여주는 UI가 필요하지만 웹 개발(HTML/CSS/JS)은 별개 작업이다.  
Streamlit은 **파이썬 코드만으로 웹 앱을 만드는 도구**다.

```python
import streamlit as st

text = st.text_area("판결문 입력")
if st.button("익명화"):
    result = model.predict(text)
    st.write(result)
```

HTML 한 줄 없이 웹 앱이 된다.

#### Hugging Face Spaces란

Streamlit 앱을 **무료로 인터넷에 올릴 수 있는 플랫폼**이다.

```
로컬에서 개발한 app.py
        ↓ 업로드
Hugging Face Spaces → 누구나 브라우저로 접속 가능한 URL 생성
```

포트폴리오에서 "여기서 직접 써보세요" 링크로 활용한다.
