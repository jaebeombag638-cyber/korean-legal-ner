## NER (Named Entity Recognition, 개체명 인식)

텍스트에서 "의미 있는 고유 명사"를 찾아내서 분류하는 작업.

### 예시

```
"피고 최도선은 2021년 3월 부산시 중구에서 사건을 일으켰다."
```

NER 결과:

| 텍스트 | 태그 | 의미 |
|--------|------|------|
| 최도선 | PER | 사람 (Person) |
| 2021년 3월 | DAT | 날짜 (Date) |
| 부산시 중구 | LOC | 장소 (Location) |

### 이 프로젝트에서 NER의 역할

익명화를 자동화하려면 먼저 **어디에 개인정보가 있는지 탐지**해야 한다. 그게 NER의 역할이다.

```
탐지 (NER)               →   교체 (익명화)
"최도선"    = PER        →   "증인-1"
"부산시 중구" = LOC      →   "부산광역시 중구 A"
"2021년 3월" = DAT       →   "2021년 XX월"
```

NER이 틀리면 (못 찾거나 잘못 분류하면) 익명화도 틀린다. 전체 성능이 NER 정확도에 달려 있다.

### 왜 KLUE-BERT를 쓰는가

BERT는 문장 전체 문맥을 보고 판단한다. 단순 규칙(예: "2글자면 이름")보다 훨씬 정확하다.

예: "이사"가 사람 이름인지 행위인지 — 앞뒤 문맥 없이는 판단 불가. BERT는 문맥을 보고 구분한다.

## BIO 태그

NER 결과를 텍스트에 붙이는 표기 방식. 토큰(단어 조각)마다 태그를 하나씩 붙인다.

### B, I, O 의미

| 태그 | 의미 | 언제 붙이나 |
|------|------|------------|
| **B**-PER | Begin | 개체명의 **첫 번째** 토큰 |
| **I**-PER | Inside | 개체명의 **두 번째 이후** 토큰 |
| **O** | Outside | 개체명이 **아닌** 토큰 |

### 예시

```
문장: "피고 최도선은 부산시에 살았다"

토큰:  피고   최     ##도   ##선  ##은  부산   ##시  ##에  살   ##았  ##다
태그:   O    B-PER  I-PER  I-PER   O   B-LOC  I-LOC   O    O    O    O
```

### 왜 B와 I를 구분하나

구분 안 하면 붙어있는 두 개체를 하나로 착각한다.

```
"홍길동 김철수" (사람 2명)

구분 없을 때:  PER PER PER PER PER       → 하나로 읽힘  ❌
BIO 사용 시:   B-PER I-PER I-PER B-PER I-PER  → 두 개로 읽힘  ✓
```

### 이 프로젝트에서 BIO가 만들어지는 흐름

```
JSON span 정보
    ↓
char_labels 배열 (문자 하나하나에 B/I/O 부여)
    ↓
BERT 토크나이저로 subword 분리
    ↓
token_labels 배열 (토큰 하나하나에 정수 레이블 부여)
    ↓
모델 학습 입력
```

`section_to_bio` 함수가 이 흐름 전체를 담당한다.

## Subword 토크나이저

### 왜 단어 단위로 자르지 않나

단어 단위로 자르면 모르는 단어가 너무 많아진다.

```
"최도선"  → 사전에 없음 → [UNK]  ❌
"홍길동은" → 사전에 없음 → [UNK]  ❌
```

한국어는 조사·활용 변형이 많아 모든 형태를 사전에 담는 게 불가능하다.

### Subword 방식

자주 나오는 조각은 그대로 두고, 드문 단어는 더 작은 조각으로 쪼갠다.

```
"최도선"  → ["최", "##도", "##선"]
"부산시"  → ["부산", "##시"]
"판결문"  → ["판결", "##문"]
"피고"    → ["피고"]           ← 자주 나오니 그대로
```

`##`은 "이 조각은 앞 조각에 이어붙는다"는 표시다.

### BIO 태그와의 연결

"최도선"이 3개 토큰으로 쪼개지면 태그를 3개 붙여야 한다.

```
토큰:    최         ##도        ##선
offset: (259, 260)  (260, 261)  (261, 262)
```

코드는 `offset_mapping`으로 각 토큰의 시작 문자 위치를 알아내고, 그 위치의 `char_labels`를 가져온다.

```python
# char_labels[259] = "B-PER"  (span 시작)
# char_labels[260] = "I-PER"
# char_labels[261] = "I-PER"

for token_start, token_end in encoding['offset_mapping']:
    token_labels.append(label2id[char_labels[token_start]])

# 결과:
# "최"   → char_labels[259] → B-PER ✓
# "##도" → char_labels[260] → I-PER ✓
# "##선" → char_labels[261] → I-PER ✓
```

### 특수 토큰 [CLS], [SEP]

BERT는 문장 앞뒤에 특수 토큰을 자동으로 붙인다.

```
입력:  최도선은 부산시에 살았다
토큰: [CLS] 최 ##도 ##선 ##은 부산 ##시 ##에 살 ##았 ##다 [SEP]
```

이 토큰들은 실제 텍스트가 아니라 `offset = (0, 0)`으로 표시된다. 코드에서 `-100`을 붙여 loss 계산에서 제외한다.

```python
if token_start == 0 and token_end == 0:
    token_labels.append(-100)   # [CLS], [SEP] → 학습 무시
```
