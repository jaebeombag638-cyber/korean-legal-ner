## 익명화 규칙 → NER 태그 매핑

EDA 첫 산출물로 이 매핑표를 완성해야 학습이 가능하다.

| 규칙 | 탐지 대상 | 처리 방법 | NER 태그 | 예시 |
|------|-----------|-----------|----------|------|
| R1 | 개인 이름(실명) | 역할 지시어로 대체 | **PER** | `최대진` → `증인-1` |
| R2 | 주민등록번호 | 완전 삭제/마스킹 | **ID** | `800101-1234567` → 삭제 |
| R3 | 상세 주소 | 시·군·구 + 알파벳 | **LOC** | `부산시 중구 광복로` → `부산광역시 중구 A` |
| R4 | 사건 날짜 | 연도만 남기고 마스킹 | **DAT** | `1961.12.20.` → `1961. XX. XX.` |
| R7 | 기업명·기관명·법원명 | 조직-N으로 대체 | **ORG** | `대전 지방법원` → `조직-2` |
| R8 | 사업자등록번호 | 삭제 | **ID** | `123-45-67890` → 삭제 |

**R1 주의사항**: 판사·검사·변호사 등 사법인물은 비식별 처리 제외. 전처리 코드에서 예외 처리 필요.  
**실제 데이터의 method**: 모든 annotation이 `"GENERALIZE"` 방식 사용.

### 1단계: EDA (2~3일)

**목표**: 규칙 분포 확인 → NER 태그 매핑표 확정 → 샘플링 전략 결정

```python
import json, glob
from collections import Counter

# 전체 파일 로드
files = glob.glob('/content/drive/MyDrive/판결문익명화/라벨링데이터/**/*.json', recursive=True)
print(f"총 파일 수: {len(files)}")

# rules_triggered 분포
rule_counter = Counter()
entity_counter = Counter()

for f in files:
    data = json.load(open(f))
    for ann in data.get('annotations', []):
        rule_counter[ann['rules_triggered']] += 1
        entity_counter[ann['original_text'][:1]] += 1  # 첫 글자로 이름 패턴 확인

print(rule_counter.most_common(10))
```

**확인할 것**:
1. `rules_triggered` 별 annotation 수 (R1이 얼마나 많은지, R4가 얼마나 많은지)
2. 분야(caseClass)별 annotation 수 분포
3. 문서당 평균 annotation 수
4. 섹션 텍스트 평균 길이 (BERT 최대 512 토큰 초과 여부)

**산출물**: rules_triggered → NER 태그 매핑표 (실데이터로 검증)

### 2단계: 전처리 → BIO 변환 (3~4일)

**핵심 주의사항**: span이 섹션 내 상대 위치이므로, 섹션별로 독립 처리한다.

```python
from transformers import AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("klue/bert-base")

# 규칙 → NER 태그 매핑
RULE_TO_TAG = {
    "R1": "PER",
    "R2": "ID",
    "R3": "LOC",
    "R4": "DAT",
    "R7": "ORG",
    "R8": "ID",
}

def section_to_bio(section_text, annotations_in_section):
    """
    섹션 텍스트 + 해당 섹션의 annotations → BIO 태그 리스트
    
    주의: span은 character 기준, BERT는 subword 토큰 기준
    → character offset → token offset 변환 필요
    """
    # 1. 문자 단위 레이블 초기화
    char_labels = ["O"] * len(section_text)
    
    # 2. annotation마다 해당 위치에 레이블 부여
    for ann in annotations_in_section:
        tag = RULE_TO_TAG.get(ann['rules_triggered'], "O")
        for span in ann['span']:
            start, end = span['start'], span['end']
            char_labels[start] = f"B-{tag}"
            for i in range(start + 1, end):
                char_labels[i] = f"I-{tag}"
    
    # 3. 토크나이저로 분리 후 토큰 레벨 레이블 매핑
    encoding = tokenizer(section_text, return_offsets_mapping=True, add_special_tokens=False)
    token_labels = []
    for token_start, token_end in encoding['offset_mapping']:
        token_labels.append(char_labels[token_start])
    
    return encoding['input_ids'], token_labels

# 검증: original_text가 실제로 span 위치에 있는지 확인
def verify_span(section_text, ann):
    for span in ann['span']:
        extracted = section_text[span['start']:span['end']]
        assert extracted == ann['original_text'], \
            f"span 불일치: '{extracted}' != '{ann['original_text']}'"
```

**검증 단계 (필수)**: 전처리 후 샘플 100건 무작위 추출해 original_text가 span에서 올바르게 추출되는지 확인. 이게 통과해야 다음 단계로 진행.

**train/val/test 분리**:
- 일반판결문: 12,000 / 1,500 / 1,500
- 1-2-최종심: 4,800 / 600 / 600
- 분야 비율 유지한 stratified split 권장

### 3단계: 모델 학습 (3~5일)

```python
from transformers import AutoModelForTokenClassification, TrainingArguments, Trainer

# NER 레이블 정의
LABELS = ["O", "B-PER", "I-PER", "B-ORG", "I-ORG", "B-LOC", "I-LOC", "B-DAT", "I-DAT", "B-ID", "I-ID"]
label2id = {l: i for i, l in enumerate(LABELS)}
id2label = {i: l for i, l in enumerate(LABELS)}

model = AutoModelForTokenClassification.from_pretrained(
    "klue/bert-base",
    num_labels=len(LABELS),
    label2id=label2id,
    id2label=id2label,
)

training_args = TrainingArguments(
    output_dir="./results",
    num_train_epochs=3,
    per_device_train_batch_size=16,
    per_device_eval_batch_size=32,
    learning_rate=3e-5,
    evaluation_strategy="epoch",
    save_strategy="epoch",
    load_best_model_at_end=True,
)
```

**실험 계획** (총 3회, 컴퓨팅 절약 고려 약 30~45분 예상):
| 실험 | learning_rate | epoch | 특이사항 |
|------|--------------|-------|---------|
| Run1 | 3e-5 | 3 | baseline |
| Run2 | 5e-5 | 5 | 더 많은 epoch |
| Run3 | 2e-5 | 3 | 낮은 lr |

### 5단계: 데모 앱 → Streamlit (2~3일)

**Streamlit 선택 이유**: Hugging Face Spaces에 무료 배포 가능, 링크 하나로 포트폴리오 제출 가능

```python
# app.py
import streamlit as st
from transformers import pipeline

@st.cache_resource
def load_model():
    return pipeline("ner", model="./best_model", aggregation_strategy="simple")

ner = load_model()

st.title("한국어 판결문 개인정보 익명화")
text = st.text_area("텍스트 입력", height=200)

if st.button("익명화 실행"):
    results = ner(text)
    # 탐지된 entity 하이라이트 + 교체 텍스트 표시
```

**기능**:
1. 텍스트 입력 → 탐지된 entity 색상별 하이라이트
2. 익명화된 결과 텍스트 출력
3. 탐지 entity 목록 표 (태그, 원문, 대체 텍스트)
