## 프로젝트 한줄 요약
AI Hub 판결문 익명화 데이터로 KLUE-BERT NER 모델을 파인튜닝해,
한국어 법률 문서의 개인정보(이름·주소·날짜·기관명)를 자동 탐지·교체하는 시스템을 구축한다.

공식 벤치마크는 GPT-OSS-120B(QLoRA)로 **F1 0.9589** 달성.
이 프로젝트는 **경량 모델(KLUE-BERT)로 얼마나 근접할 수 있는지** 검증하는 것이 핵심 질문.
목표 F1: **0.80 이상** (한계점을 솔직히 분석하는 것이 포트폴리오 개선)

### JSON 구조 (핵심만)
```json
{
  "info": { "caseClass": "가사", "annotation_count": 14 },
  "sections": [
    { "section_id": "S9", "text": "...성립을 다퉈이 있다...", "char_count": 1759 }
  ],
  "annotations": [
    {
      "annotation_id": "A1",
      "section_id": ["S9"],
      "span": [{ "start": 259, "end": 262 }],
      "original_text": "최도선",
      "method": "GENERALIZE",
      "replacement_text": "증인-1",
      "link_entity_id": "E-250844",
      "rationale": "사건 인물 실명은 직접식별자이므로 일반화 처리",
      "rules_triggered": "R1"
    }
  ]
}
``` 

> **데이터 구조 주의사항 3가지**
> 1. `span`은 **섹션 텍스트 내 상대 위치** (문서 전체 기준 아님)
> 2. `span.end`는 **exclusive** — `text[start:end]`가 `original_text`와 일치 (Python slice 기준)
> 3. `annotations[].section_id`는 **배열 타입** `["S9"]` (공식 스키마 표에는 string이라고 쓰여 있지만 실제 데이터는 배열)

### 1단계: EDA (2~3일)

**목표**: 규칙 분포 확인 → NER 태그 매핑표 확정 → 샘플링 전략 결정

```python
import json, glob
from collections import Counter

BASE = "data/12.판결서 익명처리 데이터/3.개방데이터/2.데이터(NIA)"

# 전체 라벨링 파일 로드 (Training + Validation)
files = glob.glob(f'{BASE}/**/02.라벨링데이터/**/*.json', recursive=True)
print(f"총 파일 수: {len(files)}")  # 예상: 18,900

rule_counter = Counter()
method_counter = Counter()

for f in files:
    data = json.load(open(f, encoding='utf-8'))
    for ann in data.get('annotations', []):
        rule_counter[ann['rules_triggered']] += 1
        method_counter[ann['method']] += 1

print("규칙별 분포:", rule_counter.most_common())
print("method 종류:", method_counter.most_common())
```

**확인할 것**:
1. `rules_triggered` 별 annotation 수 — **R5, R6가 실제로 나오는지, 어떤 텍스트인지** (NER 태그 결정에 필수) / **R9, R10은 PDF 가공 기준에서 R1~R8만 언급되므로 미등장 예상, 확인만**
2. `method` 값 종류 — `TOKENIZE`가 실제로 나오는지
3. 분야(caseClass)별 annotation 수 분포
4. 문서당 평균 annotation 수
5. 섹션 텍스트 평균 길이 (BERT 최대 512 토큰 초과 여부 → 512 초과하면 truncation 전략 필요)

**산출물**: rules_triggered → NER 태그 매핑표 (R5, R6, R9, R10 포함 완성본)

### 2단계: 전처리 → BIO 변환 (3~4일)

**핵심 주의사항**:
- `span`이 섹션 내 상대 위치이므로, 섹션별로 독립 처리한다
- `annotations[].section_id`는 **배열**(`["S9"]`)이므로 리스트 처리 필수
- `span.end`는 exclusive (end 위치 문자는 포함 안 됨)

```python
from transformers import AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("klue/bert-base")

# 규칙 → NER 태그 매핑 (EDA 후 R5, R6, R9, R10 채울 것)
RULE_TO_TAG = {
    "R1": "PER",
    "R2": "ID",
    "R3": "LOC",
    "R4": "DAT",
    "R7": "ORG",
    "R8": "ID",
    # R5, R6, R9, R10: EDA 결과 보고 추가
}


def build_section_annotation_map(data):
    """JSON 1개에서 section_id → [annotation 리스트] 매핑 생성.
    section_id가 배열 타입이므로 순회 처리.
    """
    mapping = {s['section_id']: [] for s in data['sections']}
    for ann in data.get('annotations', []):
        for sid in ann['section_id']:   # 배열이므로 for문
            if sid in mapping:
                mapping[sid].append(ann)
    return mapping


def section_to_bio(section_text, annotations_in_section):
    """
    섹션 텍스트 + 해당 섹션의 annotations → (token_ids, BIO 태그 리스트)

    span은 character 기준, BERT는 subword 토큰 기준
    → character offset → token offset 변환 필요
    """
    # 1. 문자 단위 레이블 초기화
    char_labels = ["O"] * len(section_text)

    # 2. annotation마다 해당 위치에 레이블 부여
    for ann in annotations_in_section:
        tag = RULE_TO_TAG.get(ann['rules_triggered'], "O")
        if tag == "O":
            continue  # 미정 규칙은 일단 O 처리, EDA 후 채울 것
        for span in ann['span']:
            start, end = span['start'], span['end']
            char_labels[start] = f"B-{tag}"
            for i in range(start + 1, end):
                char_labels[i] = f"I-{tag}"

    # 3. 토크나이저로 분리 후 토큰 레벨 레이블 매핑
    encoding = tokenizer(section_text, return_offsets_mapping=True, add_special_tokens=False)
    token_labels = []
    for token_start, token_end in encoding['offset_mapping']:
        token_labels.append(char_labels[token_start])

    return encoding['input_ids'], token_labels


def verify_span(section_text, ann):
    """전처리 검증용: original_text가 span 위치에 실제로 있는지 확인."""
    for span in ann['span']:
        extracted = section_text[span['start']:span['end']]
        assert extracted == ann['original_text'], \
            f"span 불일치: '{extracted}' != '{ann['original_text']}'"
```

**검증 단계 (필수)**: 전처리 후 샘플 100건 무작위 추출해 `verify_span`이 100% 통과하는지 확인. 이게 통과해야 다음 단계로 진행.

**train/val/test 분리**:
- Training 폴더 → train 그대로 사용 (일반판결 12,000 / 1-2-최종심 4,800)
- Validation 폴더 → 절반씩 분리 (랜덤 시드 고정)
  - 일반판결: val 750 / test 750
  - 1-2-최종심: val 300 / test 300
- 분야 비율 유지한 stratified split 권장 (Validation 분리 시)

### 3단계: 모델 학습 (3~5일)

```python
from transformers import AutoModelForTokenClassification, TrainingArguments, Trainer

# NER 레이블 정의 (EDA 후 R5/R6/R9/R10에 해당하는 태그 추가 가능)
LABELS = ["O", "B-PER", "I-PER", "B-ORG", "I-ORG", "B-LOC", "I-LOC", "B-DAT", "I-DAT", "B-ID", "I-ID"]
label2id = {l: i for i, l in enumerate(LABELS)}
id2label = {i: l for i, l in enumerate(LABELS)}

model = AutoModelForTokenClassification.from_pretrained(
    "klue/bert-base",
    num_labels=len(LABELS),
    label2id=label2id,
    id2label=id2label,
)

training_args = TrainingArguments(
    output_dir="./results",
    num_train_epochs=3,
    per_device_train_batch_size=16,
    per_device_eval_batch_size=32,
    learning_rate=3e-5,
    evaluation_strategy="epoch",
    save_strategy="epoch",
    load_best_model_at_end=True,
)
```

**실험 계획** (총 3회, T4 기준 약 30~45분/회):
| 실험 | learning_rate | epoch | 특이사항 |
|------|--------------|-------|---------|
| Run1 | 3e-5 | 3 | baseline |
| Run2 | 5e-5 | 5 | 더 많은 epoch |
| Run3 | 2e-5 | 3 | 낮은 lr |

### 5단계: 데모 앱 → Streamlit (2~3일)

**Streamlit 선택 이유**: Hugging Face Spaces에 무료 배포 가능, 링크 하나로 포트폴리오 제출 가능

```python
# app.py
import streamlit as st
from transformers import pipeline

@st.cache_resource
def load_model():
    return pipeline("ner", model="./best_model", aggregation_strategy="simple")

ner = load_model()

st.title("한국어 판결문 개인정보 익명화")
text = st.text_area("텍스트 입력", height=200)

if st.button("익명화 실행"):
    results = ner(text)
    # 탐지된 entity 하이라이트 + 교체 텍스트 표시
```

**기능**:
1. 텍스트 입력 → 탐지된 entity 색상별 하이라이트
2. 익명화된 결과 텍스트 출력
3. 탐지 entity 목록 표 (태그, 원문, 대체 텍스트)
